Learning Objectives

At the end of this lesson you should have set up Git and Node.js and you will be able to:

  • Set up a Git repository and perform basic Git operations
  • Set up and use online Git repositories
  • Use Node-based modules to perform basic operations.

Download and Install Git

  • Download the Git installer for your specific operating system.
  • Follow the installation steps as you install Git using the installer.
  • Git documentation for installing Git.

Basic Git Commands

Open a cmd window or terminal on your computer. To check that Git has installed and is available:

git --version

Finish Git configuration with the following commands:

git config --global user.name "Your Name"

git config --global user.email your "email address"

Check configuration with:

git config --list

Responsive Design and the Bootstrap Grid System

Learning Objectives

Overview of responsive web design with a mobile first appraoch. Introduction to the Bootstrap grid system. Implement Bootstrap grid in order to make it responsive with fluid images and media queries. At the end of this lesson, you will be able to:

  • Understand the reasons for using responsive web design in a web project
  • Use the Bootstrap grid system to design responsive websites
  • Add your own custom CSS classes to a Bootstrap based web project

Key feautures and classes

  • Outer div with class="container" or,
  • class="container-fluid" for a responsive container with full device screen width
  • Inner div with class="row" This will create a row with 12 equal Bootstrap columns

Setting Bootstrap Grid Column Classes and Corresponding Screen Sizes

  • .col // extra small screen sizes below // column full width
  • .col-sm // small screen size 576px // max container width 540px
  • .col-md // screen size 768px // max container width 720px
  • .col-lg // screen size 992px // max container width 960px
  • .col-xl // screen size 1200px // max container width 1140px

Specifying Layout Size and Alignment`

  • use screen size specifier -sm, -md, etc to declare rules for those screen sizes
  • Append a numeric value from 1-12 to span the content by that many column widths
  • .col-sm-5 // append -5 to -sm screen class for content that spans two coloumn widths on small screen size

    • Declare multiple classes to specify layout for alternative screen sizes, for example:

  • class="col-12 col-sm-6"

    • col-12 = entire width on xs screens // col-sm-6 = half-width on sm screens and above.
